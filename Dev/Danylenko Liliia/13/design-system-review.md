# Рев'ю дизайн-системи

## Дата: 13 листопада
## Учасники: Команда розробки + дизайнери

---

## 1. Кольори для департаментів

### Проблема:
- Захардкоджені значення кольорів окремо для Light Theme і Dark Theme
- В системі є тільки один основний колір для департаменту, який зберігається в lib
- Неможливо хардкодити кольори всередині кожного мікросервісу

### Рішення:
- Використовувати opacity для hover ефектів
- Додавати opacity в хеш-форматі (наприклад, дві шістки, дві вісімки)
- Для active стану використовувати brightness для затемнення
- Колір зберігається в одному місці (lib), змінюється там - змінюється всюди
- Не можна захардкодити інший колір саме для департаментів, тільки гратися з opacity та brightness

---

## 2. Кнопки

### Проблема:
- Створені окремі компоненти: Primary, Success, Delete
- Outline модель може бути тільки синім
- Нелогічна структура компонентів

### Рішення:
- Об'єднати в один компонент з варіантами: `contained` (заповнена) або `outline` (з бордером)
- Додати тип `Ghost` (прозорий бекграунд, текст, при hover міняється бекграунд)
- Колір можна придавати будь-який (primary синій, success зелений тощо)
- Використовувати один компонент з різними варіантами замість окремих компонентів

### Додаткові питання:
- Кнопки ще не оновлені, треба точково сідати і дивитися
- Буде синенька без градієнту і з білим текстом
- Глобальне налаштування кнопки, бо всюди робили по-різному в різні етапи

---

## 3. Breadcrumbs

### Проблема:
- Не дороблена логіка для неактивних елементів
- Немає розписаного поведінки при hover
- Немає стилів при наведенні (колір, underline, курсор)

### Рішення:
- Додати стилі при hover на неактивні елементи
- Елементи мають працювати як посилання
- При hover: зміна кольору, поява underline, зміна курсора на pointer
- При кліку - перехід на сторінку

---

## 4. Інпути

### Проблема:
- При фокусі залишений outline бордер
- Непотрібна підсвітка outline

### Рішення:
- Прибрати outline при фокусі
- Залишити тільки підсвітку бордера в інший колір замість сірого

---

## 5. Дропбокси (Select)

### Проблеми:
1. **Автокомпліт:**
   - Пропущена логіка автокомпліту
   - В select-ах, особливо в multiple, може бути дуже багато варіантів
   - Потрібен пошук або обмеження висоти

2. **Відображення інпуту:**
   - Немає самого інпуту для дропбоксу
   - Не зрозуміло, як він має відображатися як звичайний інпут
   - Не зрозуміло, як відображати вибрані опції (multiple або single) в самому інпуті

### Рішення:
- Додати пошук для автокомпліту
- Додати стилі для обмеження висоти дропбоксу
- Розписати логіку відображення інпуту та вибраних опцій

## 8. Фільтри / Карточки фільтрів

### Проблема:
- При hover прибраний бордер, хоча в дефолті/активі він є
- Не зрозуміло: бордер стає прозорим чи кольору фону?
- Якщо прибрати бордер, карточка буде стрибати (CSS поведінка)

### Рішення:
- Чітко визначити поведінку бордера при hover
- Або бордер стає прозорим, або кольору фону (щоб не було стрибків)
- Не прибирати бордер повністю

## 16. Cover-картинки

### Проблема:
- Не зрозуміло, як розміщати картинку в блоці
- Потрібні деталі по позиціонуванню

### Рішення (з обговорення):
- **Для курсів:** позиція `right center` (права сторона по центру)
- **Для уроків:** позиція `center center` (центр по центру)
- Встановити на програмному рівні позицію картинки
- Всі картинки будуть вирівняні однаково
- За рахунок різної кількості контенту та ширини пристроїв - найкращий варіант `center center`

### Додаткові питання:
- Оновити відступи від зображення
- Градієнт на фоні - поки немає, треба придумати заглушку
- Відчуття, що модуль ще завантажується

## 17. Алерти

### Проблема:
- В більшості проєктів використовуються тости, а не алерти
- Не продумано:
  - На всю ширину чи в куті?
  - Позиція відображення
  - Чи має бути хрестик?
  - Чи має бути полосочка, що зникне через час?

### Рішення:
- Компонент взагалі не продуманий
- Можливо в Markdown прописано більше
- Треба доробити або замінити на тости

## 26. Архітектура рішення

### Підхід:
- Створити свій проєкт з компонентами (як з MUI працюємо)
- Витягуємо компоненти (акордеон, все інше)
- Можемо міняти стилі для цих компонентів
- Можемо міняти кольори

### Приклад: "MuiUikid"
- **Карточка курсу:**
  - Витягується в онлайн-академії
  - Витягується в мікросервісі для рекрутерів (m2e)
  - Витягується в талантах
  - Витягується в юзерах
  - Всюди однакові дані по курсу

### Принцип:
- Компонент витягується з одного місця
- Кольори міняємо в бібліотеках
- Заокруглення міняємо в бібліотеках
- Можемо майже все поміняти
- Але є готові рішення, які можемо не міняти
