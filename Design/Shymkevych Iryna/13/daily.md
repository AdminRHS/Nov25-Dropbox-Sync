### Instructions:
Email HTML is notoriously tricky because different email clients render HTML differently. Start by testing in the most common clients: Gmail (most restrictive), Outlook (uses Word rendering engine), and Apple Mail (most standards-compliant). Send actual test emails to yourself rather than just viewing HTML in browsers. Look for: broken layouts, missing images, incorrect fonts, spacing issues, or non-clickable links. Common problems include: CSS not supported (use inline styles), background images not displaying (Outlook doesn't support them), or responsive design breaking. Use AI to help fix these issues, but be prepared to manually adjust code. Email HTML often requires table-based layouts instead of modern flexbox/grid. Test thoroughly on mobile since many people read emails on phones.

## Task 3: Optimize HTML email code for compatibility

### Steps:
1. Use AI to analyze current HTML code for email compatibility issues
2. Convert any modern CSS to inline styles
3. Replace flexbox/grid layouts with table-based structures if needed
4. Ensure all images have alt text and proper dimensions
5. Add email-safe fonts with fallbacks
6. Test background colors work in all clients
7. Verify buttons are coded as bulletproof (table-based, not just styled links)
8. Make changes manually to optimize code structure
9. Remove any unsupported CSS properties
10. Add email-specific meta tags and doctype
11. Minimize code bloat while maintaining functionality
12. Document all compatibility workarounds used

### Resources and Links:
- [Newsletter HTML Code](https://docs.google.com/document/d/1Wn8DJquejx8Mjm5tVkYmG-ZfTaMGnao7DZHwHCwzq50/edit?usp=sharing)
- [Email HTML Best Practices](Campaign Monitor, Mailchimp guides)
- [Can I Email](caniemall.com - email client compatibility reference)

### Instructions:
Email HTML is stuck in the past - you need to write code like it's 1999. Start by ensuring all CSS is inline (style="property: value;" on each element) because many email clients strip out <style> tags. Use tables for layout, not divs with flexbox/grid. Outlook especially requires table-based layouts. For buttons, use "bulletproof" button code (VML for Outlook + regular HTML). Stick to web-safe fonts: Arial, Georgia, Verdana, etc., with fallbacks. Avoid: background images (except as background-color), video embeds, JavaScript, forms, or complex CSS. Test that images have explicit width/height. Use AI to help convert modern HTML to email-compatible code, but review carefully - AI might suggest modern techniques that don't work in email. The 600px width standard comes from optimal reading width and compatibility with most email clients at default zoom.

## Task 4: Create responsive versions for mobile

### Steps:
1. Participating in testing on actual mobile devices
2. Take note of any layout issues on smaller screens
3. Use media queries to adjust layout for mobile (if email client supports them)
4. Ensure text remains readable without zooming
5. Make buttons large enough for finger taps (minimum 44x44px)
6. Test images scale properly and don't overflow
7. Verify navigation/links are easily clickable on mobile
8. Check that tables stack appropriately on narrow screens
9. Ensure social media icons remain visible and functional
10. Test in Gmail mobile app specifically (most common)
11. Document mobile-specific adjustments made
12. Create fallback design for clients that don't support media queries

### Resources and Links:
- [Newsletter Code](https://docs.google.com/document/d/1ijCCC_7259TksR8NBxSeSuaa-N079CDLOP1vPGZTDwk/edit?usp=sharing)
- [Mobile Email Testing Guide](Google Drive)
- Actual mobile devices or browser dev tools

### Instructions:
Most people read emails on mobile, so this is critical. The challenge: not all email clients support media queries (responsive CSS). Gmail app does, but many don't. Your approach: use fluid/hybrid design that works even without media queries, then enhance with media queries where supported. Use percentage widths instead of fixed pixels where possible. Stack columns vertically for mobile. Make text at least 14px (16px better) so it's readable without zooming. Buttons should be minimum 44x44px for easy tapping with fingers. Test on actual phones if possible - iOS Mail and Gmail app are most important. Send yourself test emails and open on your phone. Check: Does text wrap properly? Are images scaled correctly? Can you tap links easily? Does content flow logically top-to-bottom? Some email clients will ignore your media queries, so the "desktop" version should still be usable on mobile as a fallback.
