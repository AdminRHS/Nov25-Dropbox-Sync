- Example: Artem Skichko portrait (row 103)

### Instructions:

**Documentation Structure:**

**1. Portrait Creation Standard Operating Procedure**

**Purpose**: Ensure consistency across all employee portraits for the CRM database

**Technical Specifications:**
- Frame dimensions: 1000x1000 pixels
- Source image: Generated or provided portrait, cropped at waist
- Final crop: Shoulders (professional headshot style)
- Export format: WebP
- File naming convention: [FirstName]_[LastName]_portrait.webp

**Step-by-Step Workflow:**

1. **Prepare Source Image**
   - Obtain AI-generated portrait or professional photo
   - Ensure image is cropped at waist level minimum
   - Verify image quality is sufficient for cropping

2. **Create Figma Frame**
   - Open Figma design workspace
   - Create new frame: 1000x1000px
   - Name frame: "[Employee Name] Portrait"

3. **Import and Position**
   - Paste PNG portrait into 1000x1000px frame
   - Position image so shoulders fill frame appropriately
   - Ensure face is centered and properly proportioned
   - Final visible area: Shoulders and up (headshot style)

4. **Export Settings**
   - Select frame in Figma
   - Export as PNG first (for quality check)
   - Convert PNG to WebP format
   - Verify file size is reasonable (typically 50-200KB)

5. **Upload to CRM**
   - Access employee database spreadsheet
   - Locate employee row
   - Upload WebP file to designated column
   - Verify image displays correctly
   - Mark portrait as complete

**Quality Checklist:**
- [ ] Frame is exactly 1000x1000px
- [ ] Portrait is centered and properly cropped at shoulders
- [ ] Face is clearly visible and well-lit
- [ ] Image quality is sharp and professional
- [ ] File is converted to WebP format
- [ ] File uploaded to correct employee row in CRM
- [ ] Portrait displays correctly in database

**Tools Required:**
- Figma (for frame creation and positioning)
- PNG to WebP converter (online tool or software)
- Access to employee database spreadsheet

**Tips:**
- Keep consistent spacing around the face (not too tight, not too loose)
- Ensure similar lighting/contrast levels across all portraits
- Maintain professional appearance standards
- If source image quality is poor, request new generation/photo

**Distribution:**
- Save this document in design department knowledge base
- Share link with all team members who create portraits
- Reference this SOP when onboarding new designers

### Steps:

3. Identify which steps could be automated

### Resources and Links:
- [Current Manual Workflow Tutorial](https://drive.google.com/open?id=1W-iGSavvtrsqyo2wwOdwc1lSdG_4BuQq&usp=drive_fs)
- [Example Output](https://drive.google.com/open?id=1POrqsY0VWdFvSEevBkwHMFgAcs0cZBGz&usp=drive_fs)
- Nabla Banana (or similar) API documentation
- Runway API documentation
- Development team contacts

### Instructions:

**Current Workflow Analysis:**

**Manual Process Steps:**
1. Client wants to see their logo on clothing products
2. Designer uses Nabla Banana (AI tool) to place logo on product photo
   - Upload product image
   - Upload logo image
   - Prompt: "Place this logo on [specific clothing item]"
   - Adjust if colors clash (e.g., invert text colors)
   - Remove watermark from generated image
3. Designer uses ChatGPT to generate animation prompt for Runway
4. Designer uses Runway to animate the image
   - Upload image with logo
   - Input AI-generated prompt
   - Generate animated video
5. Designer downloads and delivers final image and video to client

**Automation Possibilities Research:**

**Option 1: Full Automation (Client Self-Service)**
- Client uploads product photo and logo through web interface
- Backend automatically processes through AI APIs
- System generates both static image and animated video
- Client receives download links

**Option 2: Semi-Automation (Designer-Assisted)**
- Designer uploads materials to automated pipeline
- System handles AI processing
- Designer reviews and approves results
- Manual adjustments if needed

**Option 3: Batch Processing**
- Designer uploads multiple product photos and logo
- System processes all variations automatically
- Designer reviews batch and selects best outputs

**Technical Research Questions:**
1. Does Nabla Banana offer an API? What are usage limits and costs?
2. Does Runway offer an API for programmatic video generation? Costs?
3. What AI alternatives exist with better API support?
4. Can logo color adjustment be automated (detecting color clashes)?
5. Can watermark removal be automated or avoided?
6. What server infrastructure is needed for processing?
7. What are typical processing times?
8. How to handle quality control?

**Development Team Consultation:**

Prepare questions for developers:
- "Is it feasible to integrate these AI APIs into the website?"
- "What would the development timeline look like?"
- "What ongoing maintenance would be required?"
- "How would we handle API costs at scale?"
- "What user interface would work best for clients?"
- "How do we ensure quality control before client sees results?"

**Cost-Benefit Analysis:**

**Costs:**
- Development time (frontend, backend, API integration)
- AI API usage fees (per image/video generated)
- Server/hosting infrastructure
- Ongoing maintenance and updates
- Quality control measures

**Benefits:**
- Client satisfaction (immediate visualization of products with their logo)
- Potential revenue increase (premium feature)
- Designer time saved (if semi-automated)
- Scalability (handle many clients simultaneously)
- Competitive advantage

1. **Executive Summary**
   - Problem statement
   - Proposed solution

2. **Current State**

7. **Recommendation**
   - Preferred option with justification

   - Success metrics

**Decision Criteria:**

- Are API costs sustainable at scale?
- Will automation maintain quality standards?
- Does it align with business priorities?
- Is there client demand for this feature?

**Mascot Style Guidelines:**

**Consistency Requirements:**
1. **Bird Species**: Each mascot must be a bird (aligns with Hummingbird primary mascot)

   - Same level of detail across all mascots

4. **Pose/Composition**:

**Bird Selection Strategy:**

Match bird characteristics to profession:
- **UI/UX Designer**: Intelligent, precise bird (owl, raven, crow)

- **Motion Designer**: Dynamic, flying bird (swift, swallow, falcon alternative)
- **Illustrator**: Artistic bird with unique patterns (kingfisher, toucan, hoopoe)
- **Brand Designer**: Distinctive, memorable bird (cardinal, blue jay, cock of the rock)
- **Web Designer**: Connected, social bird (sparrow flock theme, starling)
- **Product Designer**: Practical, problem-solving bird (corvid family)
- **Video Editor**: Storytelling bird (mockingbird, lyrebird)
- **Photo Editor**: Detail-oriented bird (eagle eye theme, hawk)
