## Project Recognition Strategies

### 1. Direct Name Match
Match exact project names
- "Remote Helpers" → **RH / Remote Helpers**
- "Design Studio" → **DGN (Design Studio)**
- "LibDev" → **LibDev / Talents Microservice**

### 2. Abbreviation Match
Recognize common abbreviations
- "RH" → **RH / Remote Helpers**
- "DGN" → **DGN (Design Studio)**
- "l-gn" → **l-gn (Lead Generation)**

### 3. Keyword Match
Match descriptive keywords
- "CRM system", "task templates" → **CRM**
- "lead generation platform", "prospecting" → **l-gn (Lead Generation)**
- "translation services" → **trn-s (Translators)**

### 4. Multi-lingual Project Names
- "Ты не сам" / "Ти не сам" → **Ty ne sam**
- "Слайди і плашки" → **RH Video Slides**
- "Портфоліо проектів" → **Portfolio of Projects**
- "Стікери на новий рік" → **RE New Year Stickers**

### 5. Contextual Clues
- Design resources mentioned → likely has Figma/Google Drive resources
- SMM content discussion → check for SMM-related projects
- Client-facing platform → likely one of the service platforms
- Internal tools → likely CRM, LibDev, or management systems

### 6. Resource-Based Identification
When specific tools/resources are mentioned:
- "Figma components" → Check for projects with Figma resources
- "Google Drive folder" → Multiple projects possible
- "SMM posts" → RH, trn-s, d-vs, of-w

## Confidence Levels

### High Confidence
- Exact match or standard abbreviation used
- Project name explicitly stated
- Unique keyword match

### Medium Confidence
- Keyword match or strong context
- Resource type matches project resources
- Department alignment suggests project

### Low Confidence
- Indirect reference or assumed from context
- Generic description without specific identifiers
- Multiple projects possible

---

## Output Format

When identifying projects, use this format:

```
**RH / Remote Helpers** - Matched from Project Directory - Confidence: High
**DGN (Design Studio)** - Matched via keyword "design services" - Confidence: Medium
**New Client Platform** - Not in Project Directory - *New project or requires manual mapping* - Confidence: Low
```

## Integration into Daily Report Prompts

### Purpose Section
Add project directory reference:
- "This prompt uses the Remote Helpers Project Directory (31+ projects) to automatically identify projects"

### Department Context Section
Include relevant projects for the specific department:
```markdown
### {Department} Projects
- Project 1 - Description - Resources
- Project 2 - Description - Resources
```

### Department Instructions Section
Add project identification step:
```markdown
**Step 1.2: Identify Projects**
- Match all mentioned projects to Project Directory
- Assign confidence levels (High/Medium/Low)
- Flag unmatched projects for manual verification
- Reference known Figma/Drive resources when project is identified
```

### Quality Standards Section
Add project identification quality criteria:
- All projects must be matched against Project Directory
- Include confidence levels for all matches
- Flag unidentified projects with contextual clues
- Reference project resources (Figma, Google Drive) when available
