## Cross-Department Patterns

### Collaboration Patterns

#### Design + Dev
- **Common Activities**: Design handoff, component implementation, design QA
- **Key Objects**: Figma files, components, specifications, mockups
- **Communication**: Design reviews, implementation questions, bug reports

#### LG + Sales
- **Common Activities**: Lead handoff, qualification verification, feedback loops
- **Key Objects**: leads, prospects, qualification criteria, handoff documentation
- **Communication**: Lead quality discussions, target refinement, win/loss analysis

#### AI + All Departments
- **Common Activities**: Automation implementation, tool training, process optimization
- **Key Objects**: prompts, workflows, documentation, training materials
- **Communication**: Opportunity identification, tool evaluation, feedback collection

#### HR + All Departments
- **Common Activities**: Recruitment, onboarding, training, performance management
- **Key Objects**: candidates, employees, job descriptions, onboarding materials
- **Communication**: Hiring needs, candidate feedback, onboarding support

### Project Alignment

#### Service Platform Projects (Design + Dev + Marketing)
- DGN, l-gn, v-mg, trn-s, smm-s, d-vs, fv-e, fv-a, of-w
- Requires coordination across branding, development, and promotion

#### Internal Systems (AI + Dev + HR)
- CRM, LibDev / Talents Microservice
- Requires technical implementation with operational input

#### Client Projects (Sales + Design + Dev)
- TipCan Solutions, Comfort Volt, AnyEmployee
- Requires project management, delivery coordination, and client communication

```javascript
      "Recruitment Activities",

    common_objects: ["candidates", "interviews", "contracts", ...],
    key_metrics: ["candidates_sourced", "time_to_hire", ...],

```

### Focus Area Templates

Each department should have templates that emphasize:
1. **Section priorities** - Which of the 21 sections are most important
2. **Object recognition** - Department-specific objects to watch for
3. **Metric extraction** - KPIs relevant to the department
4. **Activity patterns** - Common workflows and processes
5. **Collaboration points** - How this department interacts with others
